MUI

MUI is a JS libarary / framework for creating uniform User Interface elements. Uniform as in it acts and looks the same in every browser, platform, environment (touch / mouse / pointer).

If you are sick of jQuery plugins that doesn't work half the time, or doesn't understand how big libraries (Ember, Angular) works, this is for you.

Download Source

How is it different from Polymer / X-Tag ?

  • It doesn't require any experimental techonolgy (web components, shadow-dom, scoped styles).
  • Fewer polyfills (requestAnimationFrame, matchesSelector)
  • Fully customizable (namespace, monkeypatch, etc...)
  • Fully skinable
  • Extendable
  • Plays nice with other frameworks
  • Fully tested in all supported browsers

Quickstart

Just load the Javascript and CSS:

<link rel='stylesheet' type='text/css' href='http://m-ui.org/assets/default/theme.css'/>
<script type='text/javascript' src='http://m-ui.org/assets/mui.js'></script>

You are good to go, you can now use the Standard Components:

<ui-button>Button</ui-button>
<ui-checkbox></ui-checkbox>

Custom Component

In MUI every component is basically a custom HTML Element. You can create as many elements as you like and they will behave exactly as you expect. The structure of a component is fairly simple, it consits of three parts described below.

CoffeeScript Class

# A Component is a CoffeeScript class that extends UI.Abstract
# or any other component,
# and it must be created in the UI object
class UI.Modal extends UI.Abstract
  # This specifies the tagname for the component
  # but remember that tagnames are namespaced
  # so the real tagname is ui-modal
  @TAGNAME: 'modal'

  # Defiene an API
  # You can specify getters, setters and methods
  # that will be available for the component
  # and the scope will be the component
  @get 'isOpen', -> @hasAttribute 'open'

  close: ->
    return if @disabled
    @removeAttribute 'open'

  open: ->
    return if @disabled
    @setAttribute 'open', true

  toggle: ->
    return if @disabled
    @toggleAttribute 'open'

  # Initialize will run when the component is
  # inserted into the document
  initialize: ->
    document.body.appendChild(@)

CSS

ui-modal {
  position: absolute;
  opacity: 0;
  z-index: -1;
  top: 0;
  left: 0;
  bottom: 0;
  right: 0;
  background: rgba(0,0,0,0.7);
  text-align: center;
  transition: opacity 600ms, z-index 1ms 600ms;
}
ui-modal[disabled] {
  opacity: 0;
}
ui-modal[open] {
  opacity: 1;
  z-index: 9999;
  transition: opacity 600ms;
}

Use it

<ui-modal>Content</ui-modal>
<script>
  modal = document.querySelector('ui-modal')
  modal.open()
</script>

Standard Components

MUI provides the following standard components, you can try all of them in the kithcen sink page.

Name Description HTML Counterpart
UI.Button Clickable compontent that calls for action. input[type=button], button
UI.Checkbox A compontent that has two states. input[type=checkbox]
UI.Color Color picker component. input[type=color]
UI.Dropdown General dropdown component (used in UI.Select).
UI.Email A component for email input. input[type=email]
UI.Form Form component that geathers data. form
UI.Grid For multi column data.
UI.Label Contains text that describes an other component. label
UI.Modal Customizable dialog component.
UI.Option Option component. option
UI.Page Page component (used in UI.Pager).
UI.Pager Pager component, for multipage content.
UI.Radio A components that allows election from different values. input[type=radio]
UI.Range A component for setting numerical value between two values. input[type=range]
UI.Select A component for selecting one from different values. select
UI.Slider Slider component.
UI.Text A component for one line text input. input[type=text]
UI.Textarea A component for text input. textarea
UI.Toggle A compontent that has two states and displayed separatly.
UI.Tooltip Show an informative text on hover.
